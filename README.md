# 法学生的简历skill

面向法学生的简历与求职材料优化 skill。![Uploading 截屏2026-06-22 18.04.21.png…]()


这个 skill 的重点不是把一份简历“润色得更像简历”，而是把用户已有材料当作**岗位匹配证据**来处理：先判断这份简历到底是给谁看的，再决定该删什么、补什么、前置什么、深挖什么，最后在需要时直接交付 `.docx` 文件。

## 这个 skill 的特点

### 1. 不是泛泛改简历，而是先做场景分流

它会先判断用户目标属于哪一类：

- 红圈争议解决 / 诉讼 / 仲裁
- 红圈非诉 / 商事 / 交易
- 大厂法务 / 合规 / 数据合规
- 法院 / 检察院 / 法检系统
- 校园组织 / 校内申请
- 通用法学生实习 / 校招

也就是说，它不是把所有法学生简历都按一个模板改，而是先看**这份材料最终是给谁看的**。

### 2. 第一轮优先输出“修改意见”，不是直接糊一版简历

默认第一轮会先给：

1. 使用场景判断
2. 当前问题
3. 我会怎么调整
4. 进一步内容挖掘问题
5. 下一步说明

这个设计的核心是：  
很多法学生简历的问题，不是文笔不够，而是**场景不清、经历没展开、研究不贴业务、法检/校园经历没有职业化翻译**。

### 3. 会按法学生真实场景深挖，而不是问泛泛问题

比如：

- 律所争议解决会追问案由、检索、文书、证据梳理
- 大厂法务会追问业务理解、合规议题、工具、英语
- 法检经历会追问案件类型、类案检索、裁判文书、数字司法
- 校园申请会判断是否应该保留更多校园经历而弱化职业化表达

### 4. 经历不足时，不直接判死刑

如果用户实习不够，这个 skill 会尝试用：

- 法律研究
- 课程项目
- 模拟法庭
- 法检经历
- 校园材料

去补“可投递证据”，而不是只说“背景不够”。

### 5. 可以直接生成 Word / `.docx`

这个 skill 自带本地脚本，可以把整理好的简历内容直接生成 `.docx`，不依赖 `python-docx` 或额外 `pip install`。

当前链路是：

`定向简历正文 -> JSON -> .docx -> verify`

## 输出节奏

### 第一轮

输入：

- `岗位要求 + 现有简历`
- 或 `个人经历 / 素材 / 零散信息`

输出：

- 一份结构化修改意见
- 一组围绕现有材料的深挖问题

### 第二轮

用户补充后，输出：

- 定向简历草稿
- 或单段精修版本

### 第三轮

如用户要求文件交付，输出：

- 本地 `.docx` 简历文件

## Word 文件交付

内置脚本：

- `scripts/generate_resume_docx.py`
- `scripts/verify_resume_docx.py`

模板：

- `assets/resume-template.json`

默认字体策略：

- macOS：`STKaiti`
- Windows：`楷体`
- 英文：`Calibri`

默认排版目标：

- A4
- 单页优先
- 专业、紧凑、易读
- 行距和段距可为单页效果自由调整

## 仓库内容

这个仓库只保留 skill 运行需要的最小结构：

- `SKILL.md`
- `agents/`
- `scripts/`
- `assets/`
- `references/notes/`

不包含学习视频、本地截图、测试输出文件等过程材料。

## 适合谁

这个 skill 适合：

- 想投律所 / 法务 / 法检 / 校园组织的法学生
- 已经有一版简历，但不知道到底问题在哪的人
- 实习经历不强，需要把研究和课程经历转成投递材料的人
- 想直接拿到 `.docx` 简历文件的人
