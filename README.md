# 法学生的简历 skill

大家好，这是一份“专门面向法学生的简历与求职材料优化 skill“。没有接触过 skill 概念的朋友也不要灰心，下面我会一步一步教你怎么做。

**开源发心** ：我还是初年级的时候，得到了很多师兄师姐们的帮助；现在我的简历经过了多轮打磨、得到了红圈所合伙人在内的前辈的认可，也很希望能把我得到的这份帮助传递下去。但我一个人也无法帮助到所有的师弟师妹们，所以结合我自己的经验和网络上能收集到的教程做了这份 skill，希望大家都不会再被简历制作的问题所困扰。

![工作流程](law-student-resume-skill-workflow)

## 如何使用

这份 skill 可以用于市面上常见的 skill（如 codex、claude code 等等），此处推荐最简单的国产软件腾讯的 workbuddy。

### 1. 安装

在 workbuddy 官网，下载并安装完成 workbuddy。

打开 WorkBuddy 后，在对话框中输入：

```text
帮我安装这个 skill：https://github.com/Ann-planning/Lawstudent-resume-skill
```
等待 workbuddy 把 skill 安装完毕就可以啦～ skill安装好后可以进行下一步。

### 2. 提供材料

#### 情况 1：你已经有一版简历

准备：

- 一份岗位 JD
- 你当前的简历全文

#### 情况 2：你还没有完整简历

准备：

- 教育背景
- 实习经历
- 校园经历
- 法律研究 / 课程实践
- 技能证书
- 想投的大致方向

### 3. 直接输入需求

最常见的三种用法：

#### 分析现有简历

```text
这是我的简历，用 lawstudent-resume-skill帮我看看有什么问题。
```

#### 按岗位定向修改

```text
这是岗位 JD 和我的简历，用 lawstudent-resume-skill请先帮我分析问题，再告诉我怎么改。
```

#### 直接生成 Word

```text
这版内容确认了，用 lawstudent-resume-skill直接帮我生成一份 .docx 简历。
```

## 这个 skill 能做什么

它的重点不是泛泛润色，而是把你的材料当作**岗位匹配证据**来处理。

主要适用于：

- 争议解决 / 诉讼 / 仲裁
- 非诉 / 商事 / 交易
- 企业法务 / 合规 / 数据合规 / 知识产权
- 法院 / 检察院 / 法检系统
- 校园组织 / 校内申请
- 通用法学生实习 / 校招

它会优先帮你解决这些问题：

- 这份简历到底是给谁看的
- 哪些经历该保留、删除、前置或后置
- 法检、模拟法庭、课程作业、校园经历怎么转成岗位证据
- 经历不足时，怎么用研究、课程、项目做补位
- 最后如何直接导出 `.docx`

## 输出特点

### 1. 先判断场景，再改简历

它不会默认所有法学生简历都按一个模板处理，而是先判断目标岗位，再决定内容取舍。

### 2. 第一轮先输出“修改意见”

很多法学生简历的问题，不是文笔不够，而是：

- 场景不清
- 经历没展开
- 研究不贴岗位
- 法检或校园经历没有职业化翻译
- 硬技能和量化信息没有被看见

所以默认第一轮会先分析，再追问，再继续写下一版。

### 3. 会继续深挖，而不是只给套话

例如：

- 律所争议解决会追问案由、检索、文书、证据梳理
- 企业法务会追问业务理解、合规议题、工具、英语
- 法检经历会追问案件类型、类案检索、裁判文书、数字司法
- 校园申请会判断是否应保留更多校园经历

### 4. 支持直接生成 Word / `.docx`

内置脚本：

- `scripts/generate_resume_docx.py`
- `scripts/verify_resume_docx.py`

默认链路：

`定向简历正文 -> JSON -> .docx -> verify`

## 隐私与局限说明

### 隐私说明

这份 skill 的效果依赖你主动提供的材料，比如：

- 简历
- 岗位 JD
- 实习经历
- 课程、研究、证书信息

如果你担心隐私，建议在测试或演示时先做脱敏，例如：

- 手机号中间打码
- 邮箱换成示例邮箱
- 学校、单位、人名做适度匿名

仓库本身只保留运行所需的 skill 文件、脚本和说明，不包含你的个人简历内容。

### 局限说明

这份 skill 能显著提高法学生简历的岗位贴合度，但它也有边界：

1. 它不能替代真实经历本身，只能帮你改写、补位、重组表达。  
2. 它非常依赖输入质量，信息太少时输出一定会打折。  
3. 它不能保证覆盖所有团队的内部偏好，只能根据 JD 和常见招聘逻辑尽量贴近。  
4. `.docx` 文件可以本地生成，但不同电脑、不同 Word 版本下可能有轻微显示差异。  

## Word 文件交付

默认字体策略：

- macOS：`STKaiti`
- Windows：`楷体`
- 英文：`Calibri`

默认排版目标：

- A4
- 单页优先
- 专业、紧凑、易读
- 行距和段距可为单页效果灵活调整

这份 skill 适合：

- 想直接拿到 `.docx` 简历文件的人
