# 法学生简历核心判断标准

## 目标

本文件用于约束 skill 处理法学生简历时的基础判断口径。默认目标不是生成完整自传，而是输出一份围绕目标机会筛选后的单页求职材料。

## 信息排序原则

默认模块顺序如下：

1. 教育背景
2. 实习经历
3. 法律研究
4. 技能与证书
5. 校园经历或补充信息

如果研究内容与目标岗位高度相关，可以把法律研究前置。

## 简历是否有效的五项标准

- 是否能在一页内完成主要信息呈现；
- 是否能在短时间内看出目标方向；
- 是否让招聘方快速定位教育、实习、研究和硬技能；
- 是否避免无关内容占据篇幅；
- 是否把每段经历写成可追问的岗位证据。

## 内容改写原则

### 场景优先

- 不默认一份简历通投；
- 诉讼或争议解决版本，优先强调检索、文书、案件支持和事实分析；
- 非诉或商事版本，优先强调合同、尽调、交易和项目支持；
- 企业法务版本，优先强调业务理解、合规议题、英语和工具能力。

### 量化优先

- 能写数量就写数量；
- 能写对象和场景就写对象和场景；
- 能写产出就写产出。

### 证据化表达

- bullet 默认用动作动词开头；
- 尽量写成“做了什么 + 在什么场景里做 + 形成什么结果”。

### 研究内容要贴真实使用场景

- 法律研究的价值在于证明问题理解和分析能力；
- 与行业、平台、数据、知识产权、劳动或商事交易相关的研究，通常比纯理论题更适合作为求职材料的一部分。

## 取舍规则

优先删除：

- 与目标岗位无关的碎片化校园活动；
- 没有信息密度的自我评价；
- 不占优势的课程和成绩堆砌；
- 不能支撑岗位判断的奖项罗列。

优先保留：

- 与目标岗位相关性最高的实习；
- 与业务或争议场景相关的法律研究；
- 英语、法律检索、法考、AI 工具等硬技能；
- 能体现能力升级的连续经历。

## 对 skill 的直接要求

处理简历时，优先完成以下动作：

- 判断目标岗位类型；
- 识别哪些内容应保留、删除、前置或后置；
- 把普通描述改写为更强的岗位证据；
- 指出关键缺口，例如研究方向不贴岗位、量化不足、实习缺乏迭代、硬技能不可见；
- 在经历不足时，主动用法律研究和可迁移经历补位。
