# 输出契约与默认模板

## 目标

这份说明用于把 skill 的运行口径写死：什么时候先分析，什么时候再出简历，什么时候进入 `.docx` 文件交付。

它不是示例文案库，而是输出顺序和信息密度的约束。

## 默认输出节奏

### 第一轮

在以下情况，默认先输出“修改意见”而不是终稿：

- 用户输入“岗位要求 + 现有简历”；
- 用户只输入“个人经历 / 素材 / 零散信息”；
- 用户要求“帮我看看这份简历怎么改”；
- 用户要求“帮我分析一下我这份简历的问题”。

第一轮必须优先完成：

1. 使用场景判断
2. 当前问题
3. 我会怎么调整
4. 进一步内容挖掘问题
5. 下一步说明

### 第二轮

在以下情况，进入“定向简历草稿”输出：

- 用户已经回答了第一轮深挖问题；
- 用户补充了更明确的岗位方向、案件类型、业务类型、研究题目或量化信息；
- 用户明确说“先直接给我一版”。

第二轮默认输出：

1. 简短的调整前提说明
2. 定向简历正文
3. 仍待补充的信息

### 第三轮

在以下情况，进入文件交付模式：

- 用户明确要求 Word；
- 用户明确要求 `.docx`；
- 用户要求“按这个 PDF 模板排版给我一份简历文件”。

第三轮默认输出：

1. 简短说明本次文件按什么岗位版本生成
2. 先把最终简历正文落成 JSON
3. 再生成本地 `.docx` 文件
4. 如有必要，补一句说明哪些结构/渲染校验已执行、哪些未执行

## 第一轮默认模板

第一轮建议直接按下面 5 段输出。

### 1. 使用场景判断

至少写清楚：

- 我判断这份材料更适合投什么；
- 我为什么这样判断；
- 目前是否存在场景混杂或场景不清的问题。

如果用户没有明确说目标，就要写出“当前最接近哪个方向”和“还存在哪个分流不确定点”。

### 2. 当前问题

这里不要只列抽象评价。每个问题都尽量落到以下一种：

- 场景不清；
- 排序不对；
- 经历不贴岗位；
- 法检经历写得太事务；
- 研究没有业务感；
- 校园经历比重过高；
- 硬技能没有被看见；
- 缺数字、缺对象、缺产出。

### 3. 我会怎么调整

这一段必须和上一段的问题一一对应，至少说明：

- 哪些内容删除或后置；
- 哪些内容前置；
- 哪些 bullet 需要重写成“动作 + 场景 + 产出”；
- 哪些经历需要补量化；
- 哪些空白需要用法律研究、类案检索、业务研究补位。

### 4. 进一步内容挖掘问题

默认不要少于 5 个问题，也不要变成通用问卷。

问题应满足三个要求：

- 直接围绕用户已经给出的经历发问；
- 能影响下一版简历怎么写；
- 能把经历从“单位名词”挖成“岗位证据”。

优先问：

- 目标岗位到底是什么；
- 每段经历中最核心的 2 到 3 件事是什么；
- 有没有案件类型、项目类型、服务对象；
- 有没有报告、检索、文书、研究产出；
- 有没有数量、页数、项目数、案由、业务线、阅读量、参与人数等数字。

### 5. 下一步说明

这一段要把后续路径说清楚：

- 用户先补充什么；
- 我收到补充后会输出什么版本；
- 如果用户需要文件，我会再生成 `.docx`。

## 第二轮默认模板

### 1. 调整前提

先用 2 到 4 句交代：

- 当前版本是投什么场景；
- 我做了哪些取舍；
- 哪些信息还属于暂定写法。

### 2. 定向简历正文

正文默认按以下结构组织：

1. 教育背景
2. 实习经历
3. 法律研究
4. 技能与证书
5. 校园经历（仅在需要时保留）

写作要求：

- bullet 用动作动词开头；
- 每条尽量带场景和产出；
- 同类信息不要重复堆叠；
- 法检经历尽量职业化改写，不停留在事务描述。

### 3. 仍待补充的信息

如果仍有关键缺口，不要假装已经完成。应单列说明：

- 哪几处内容还缺数字；
- 哪段经历还缺案件或业务细节；
- 哪个研究主题还需要进一步贴业务。

## 当用户只给个人经历时的特殊要求

如果用户没有成稿简历，第一轮中的“当前问题”可以改写为“当前素材短板”，但结构不变，仍要包含：

- 更适合的场景判断；
- 目前最缺什么；
- 哪些素材值得保留；
- 哪些内容需要补位；
- 继续深挖问题。

## 允许的例外

如果用户明确说“别分析了，先直接给我一版”，可以在第一轮诊断后直接附一版“暂定草稿”。

但即便如此，也不能跳过：

- 使用场景判断；
- 当前问题；
- 进一步内容挖掘问题。

## 对 skill 的直接要求

skill 的稳定行为应是：

- 先分流；
- 再判断场景；
- 再做问题诊断；
- 再给调整方案和深挖问题；
- 用户补充后再出定向简历；
- 用户要求文件时再交付 `.docx`。
