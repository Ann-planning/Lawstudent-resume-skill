# 简历修改意见输出工作流

## 目标

这个 skill 在收到材料后，默认不应直接跳到“帮你润色一版简历”。它应先输出一份高质量、可执行的修改意见，再根据用户补充信息继续生成定向简历。

## 两种典型输入

### 1. 岗位要求 + 现有简历

这类输入说明用户已经有比较明确的使用场景。此时应优先：

- 判断岗位类型；
- 判断现有简历和岗位要求的贴合度；
- 指出简历当前最影响投递的问题；
- 提出进一步内容挖掘问题。

### 2. 只有个人经历 / 素材

这类输入说明用户可能还没有明确版本或没有成稿。此时应优先：

- 判断这份素材更适合投什么方向；
- 判断哪些素材值得保留；
- 判断需要补哪些研究或证据；
- 在必要时先构造一版简历结构建议，再继续追问。

## 第一轮输出的默认顺序

默认顺序应为：

1. 使用场景判断
2. 当前简历/素材的核心问题
3. 我会怎么调整
4. 进一步内容挖掘问题
5. 下一步交付说明

不要一上来就只给“润色后文本”。

## 修改意见必须包含什么

### 1. 使用场景判断

先明确这份简历到底是给谁看的，例如：

- 红圈争议解决实习
- 红圈非诉/商事
- 大厂法务/合规
- 法院/检察院
- 校园组织、学生组织、校内项目申请
- 通用求职版本

如果场景不清楚，要明确写出自己的判断和不确定点。

### 2. 简历问题诊断

不要说空泛的话，要尽量落到招聘者视角下的可见问题，例如：

- 场景不明确；
- 实习和岗位要求不贴；
- 研究方向太理论；
- 法检经历过于事务化；
- 校园经历过多但对当前目标帮助有限；
- 英语、检索、AI 工具等硬能力没有被看见；
- 经历没量化；
- 模块顺序不合理。

### 3. 调整思路

每个问题后面都要跟“我会怎么调整”，例如：

- 删什么；
- 前置什么；
- 改写什么；
- 补什么研究；
- 哪些内容改成更像岗位证据；
- 哪些内容暂时不该写。

### 4. 进一步内容挖掘

这部分是关键，不能只问泛泛的“还有别的信息吗”。

应当围绕用户已有材料、按场景继续深挖：

- 实习中到底做了什么；
- 服务对象是谁；
- 案件或业务类型是什么；
- 有没有产出报告、文章、检索结果、研究结论；
- 有没有数字、成果、反馈；
- 是否能展开讲清楚。

### 5. 下一步说明

明确告诉用户：

- 先回答这些问题；
- 回答后我会进一步把简历改成某个岗位版本；
- 如果用户要 Word 文件，最后再走 `.docx` 输出。

## 默认输出节奏

推荐节奏：

### 第一轮

输出：

- 修改意见
- 结构化深挖问题

### 第二轮

用户补充后，输出：

- 定向简历草稿
- 或单段重写版本

### 第三轮

如用户要求文件交付，再输出：

- `.docx` 简历文件

## 允许的例外

如果用户明确说“先直接给我一版”，可以在给出修改意见后，附一版“暂定草稿”。

但即便如此，也不应跳过：

- 场景判断；
- 问题诊断；
- 进一步深挖。

## 对 skill 的直接要求

这个 skill 在第一轮输出中，必须优先体现“分析、调整、深挖”三件事，而不是只做润色器。
