# 使用场景分流与深挖问题库

## 目标

这份问题库用于让 skill 在看完用户材料后，不是笼统发问，而是按使用场景和已有经历精准追问。

## 第一层：先判断这份简历给谁看

至少判断以下几类：

### 1. 律所实习 / 校招

再细分为：

- 争议解决 / 诉讼 / 仲裁
- 非诉 / 商事 / 交易

### 2. 大厂法务 / 合规 / 数据合规

### 3. 法院 / 检察院 / 法检系统

### 4. 校园组织 / 学生组织 / 校内项目申请

### 5. 暂未明确场景，只是整理基础简历

## 第二层：按场景判断内容取舍

### 律所争议解决

优先保留：

- 诉讼、仲裁、争议解决相关实习；
- 检索、文书、案件支持；
- 法检经历里的类案研究和案件分析；
- 法律研究中偏程序、裁判规则、争议议题的内容。

弱化：

- 纯事务性校园活动；
- 与争议解决无关的空泛协调类内容。

### 律所非诉 / 商事

优先保留：

- 合同、尽调、项目支持、IPO、交易文件；
- 有商业场景的研究；
- 协调沟通、项目推进。

弱化：

- 与交易无关的程序性内容；
- 过度强调诉讼风格的表达。

### 大厂法务 / 合规

优先保留：

- 业务相关研究；
- 英语和检索能力；
- 工具、AI、知识库；
- 数据合规、平台责任、消费者保护、知识产权等议题。

弱化：

- 没有业务场景的泛法律表述；
- 无法转化为企业法务能力的经历。

### 法院 / 检察院 / 法检系统

优先保留：

- 法律基础；
- 程序理解；
- 裁判文书、案件分析、检索研究；
- 对司法数字化、AI 的观察。

弱化：

- 仅靠单位抬头撑价值；
- 只写辅助事务。

### 校园组织 / 学生组织 / 校内申请

优先保留：

- 校园经历；
- 活动组织、写作、宣传、协调；
- 学生工作、竞赛、项目执行；
- 与申请目标直接相关的领导力或执行力证据。

弱化：

- 对校内申请无帮助的职业化堆砌；
- 没有必要硬塞律所或法务语言。

## 第三层：深挖问题模板

### 通用起手问题

- 这份简历是准备投什么场景？
- 是找实习、校招、校内组织申请，还是先做通用版本？
- 你最希望这份简历最终帮你拿到什么机会？

### 实习经历通用深挖

- 这段实习的具体岗位名称是什么？
- 你服务的对象是谁？是律师、法务、法院法官、检察官，还是校内老师和组织？
- 你做的最核心 2 到 3 件事是什么？
- 有没有形成过报告、文书、文章、检索成果、案例梳理？
- 有没有数量、页数、案由、项目数量、阅读量、粉丝量、参与人数等数字？
- 这些内容面试时你能讲展开吗？

### 律所争议解决深挖

- 做过什么案由？
- 有没有做过类案检索、证据梳理、备忘录、起诉状/答辩思路？
- 是否接触过客户、法官、仲裁材料或庭前准备？
- 有没有你觉得最能体现逻辑能力的一次案件支持？

### 非诉 / 商事深挖

- 做过什么类型的合同、交易文件、尽调或项目支持？
- 你支持的行业是什么？
- 有没有参与项目流程而不只是校对文本？
- 有没有可量化的项目数量、合同数量或尽调范围？

### 大厂法务 / 合规深挖

- 有没有做过与数据合规、平台责任、消费者保护、IP、跨境业务相关的研究？
- 你对目标公司的业务理解到什么程度？
- 你用过哪些检索工具、知识库、AI 工具？
- 有没有英文材料处理、翻译、英文检索或跨法域比较？

### 法检经历深挖

- 你所在的是法院还是检察院？具体是哪一级、哪类机构？
- 接触过哪些案件类型？
- 除了事务性工作，有没有主动看裁判文书或做案件梳理？
- 有没有可能补出 1 到 2 个类案研究或行业案件分析？
- 是否接触过数字法院、司法信息化、AI 辅助系统？

### 校园组织深挖

- 这份简历是给校内组织看的，还是校外招聘方看的？
- 你在组织里负责的是运营、宣传、活动、写作、外联还是统筹？
- 有没有实际活动规模、参与人数、传播数据、项目成果？
- 这类经历是否比你当前职业实习更贴这个使用场景？

## 何时建议补法律研究

满足以下任一情况，应主动建议补法律研究：

- 实习经历不足；
- 目标岗位很看重业务理解；
- 用户有法检或校园经历，但缺少职业化证据；
- 目标是大厂法务、争议解决等研究密集型岗位。

## 对 skill 的直接要求

这个 skill 的提问，不应是固定问卷，而应是：

- 先判断场景；
- 再从对应场景里选问题；
- 再围绕用户现有材料继续追问；
- 最终把答案转成简历里的证据。
